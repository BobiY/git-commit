1.初始化  git  项目

      git clone  地址 文件夹名    生成的项目名称是你指定的文件夹名如果不指定则是你的地址.git前面的文件夹名字

      .git  文件夹用于保存下下来的所有版本记录，然后中取出最新版本的文件拷贝。

2. 记录每次更新到仓库

     工作目录下的文件状态：已跟踪和未跟踪

     已跟踪的文件是指本来就已纳入版本控制管理的文件，在上次快照中有他们的记录，工作一段时间后，他们的状态可能是未更新，已修改或者放入暂存区。
   
     未跟踪文件指的是除了你拉下来的其他文件都属于未跟踪文件

3.检查当前文件状态

     git status

4.跟踪新文件

     git add  文件名（这是一个多功能的命令，根据目标文件状态的不同，此命令的效果也不同：可以用它开始跟踪新文件，或者把已跟踪的文件放到暂存区，还能用于把有冲突的文件标记为已解决状态）

     执行以后文件 再使用 git status 会显示文件已被跟踪  在 Changes to be commited 下的文件都是暂存状态。如果你此时提交，那么该文件此时此刻的版本将被留存在历史目录中。

     其实git add 的潜台词就是把目标文件快照放入暂存区域，也就是 add file into staged area，同时未曾跟踪过的文件标记为需要跟踪。这样就好理解后续 add 操作的实际意义了


5.忽略某些文件
  
    创建一个.gitignore 文件  列出来要忽略的文件模式。
   
    e.g. 
   
       *.[oa]
       *~

    文件书写规范

      1.所有空行或者以注释符号 # 开头的行都会被 Git 忽略。

      2. 反斜扛（/）说明要忽略的是目录。 e.g.   /a   表示要忽略所有文件夹wenjian
   
      3. *表示要忽略全部符合条件的wenjain   e.g.  *.a  表示要忽略所有已 .a 结尾的文件
  
      4.！表示除符合条件的文件           e.g.   !a.a 接上例，表示a.a文件不会被忽略 


     # 此为注释 C 将被 Git 忽略

     *.a       # 忽略所有 .a 结尾的文件

     !lib.a    # 但 lib.a 除外

     /TODO     # 仅仅忽略项目根目录下的 TODO 文件，不包括 subdir/TODO

     build/    # 忽略 build/ 目录下的所有文件

     doc/*.txt # 会忽略 doc/notes.txt 但不包括 doc/server/arch.txt

6.查看尚未暂存的文件更新了哪些部分  git diff
     

7.提交更新  git commit -m “本次提交声明”

8.跳过暂存区  git commit -a -m '本次提交声明'  这样就不需要使用 git add  Git会自动的把所有已经跟踪过的文件暂存起来一起提交

9.查看提交历史  git log 就可以查看提交历史 如果不加参数的话汇理出所有的更新，最近的更新排在最上面

   -p 选项展开显示每次提交的内容差异，用 -2 则仅显示最近的两次更新。

10.查看当前的远程仓库 git remote 会列出每个远成仓库的简短名字


11.Git 分支

   分支其实就是从某个提交对象往回看得历史

   新建分支 git branch 分支名字

   新建分支完成以后Git是如何知道你当前在哪个分支上工作的呢？答案很简单，Git中有一个名为 HEDA的指针，他会指向你当前所在的分支

   运行 git branch 命令，仅仅是创建了分支，但Git是不会自动的切换到这个分支中去。

   切换分支 git checkout 分支名字  这时，就切换到了新的分支上去，上文提到的HEAD指针就会指向这个分支。
